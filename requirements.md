# 纯 Web HTML5 贪吃蛇游戏 - 需求规格说明

## 1. 项目概述

- **目标**：实现一个仅使用 HTML5、CSS3 与原生 JavaScript 的贪吃蛇游戏，在浏览器中单页运行，无需后端与构建工具。
- **技术边界**：不依赖 Node.js、打包工具或任何前端框架；使用 Canvas 2D 进行绘制，逻辑与渲染均在前端完成。

## 2. 术语与定义

| 术语 | 定义 |
|------|------|
| 蛇头 | 蛇身体最前端的一节，决定移动方向与碰撞检测 |
| 蛇身 | 由多节组成的队列，除蛇头外均为蛇身 |
| 格子 | 画布按行列划分的单元，蛇与食物的坐标以格子为单位 |
| 食物 | 出现在格子上的可食用目标，蛇头与之重合即视为吃下 |
| 游戏周期 | 从「开始游戏」到「游戏结束」的一局完整流程 |

## 3. 功能需求

### FR-1 画布与网格

- 使用 HTML5 Canvas 绘制游戏区域。
- 画布按固定或可配置的行列数划分为网格（如 20×20），每格对应一个逻辑单位。
- 蛇与食物的位置均以格子坐标表示，便于碰撞与食物生成。

### FR-2 蛇的渲染与移动

- 蛇身用矩形或圆角矩形在对应格子上绘制，蛇头可与蛇身有视觉区分。
- 移动逻辑：每帧在当前方向上前进一格（新蛇头），并去掉最后一节（蛇尾）；吃下食物时本帧不去尾，蛇长度加一。
- 蛇的初始长度、初始位置与初始方向需明确（如居中、向右）。

### FR-3 食物生成

- 在画布内随机选择一个格子作为食物位置。
- 食物不得与当前蛇身（含蛇头）任意一节重叠；若无可选格子则不生成（极端情况可视为胜利或平局，可选实现）。

### FR-4 键盘输入

- 使用键盘控制蛇头方向：方向键或 WASD，上下左右四向。
- 禁止反向：例如当前向右时，不能直接改为向左；其他方向同理。
- 需做输入节流或状态锁，避免单次按键触发多次转向。

### FR-5 分数显示

- 在画布外（如上方或侧边）显示当前局分数。
- 分数规则可配置（如每吃一个食物 +10）。
- 可选：使用 localStorage 保存并显示历史最高分。

### FR-6 开始与结束界面

- **未开始时**：画布或覆盖层上显示「开始游戏」或「按某键开始」的提示。
- **游戏结束时**：显示「游戏结束」与本局最终分数，并提供「再玩一次」或「重新开始」的入口（按钮或按键）。
- 再玩一次时：蛇、食物、分数、方向均重置为初始状态。

### FR-7 难度与速度（可选）

- 可选实现：提供难度档位或随分数增加蛇的移动速度（如每 N 分提高一档速度），以增加难度。

## 4. 非功能需求

### NFR-1 兼容性

- 仅使用标准 HTML5、ES5+ 与 Canvas 2D API，不依赖第三方库。
- 目标运行环境：现代浏览器（如 Chrome、Firefox、Edge、Safari 等）。

### NFR-2 代码结构

- 游戏状态、渲染、更新与输入处理分层清晰，便于维护与扩展。
- 核心逻辑（如移动、碰撞、吃食物）与绘制逻辑分离。

### NFR-3 可读性与文档

- 关键逻辑配有简要注释。
- 本需求文档中明确「纯 Web HTML5」的边界（无 Node、无打包、无框架）。

## 5. 界面与交互

### 5.1 画布

- 画布尺寸可固定（如 400×400 像素）或按格子数计算；需保证格子为整数像素，避免错位。
- 网格线可画可不画，由实现决定。

### 5.2 按键说明

- 方向：上/下/左/右 或 W/A/S/D。
- 开始：空格或 Enter 等（与「再玩一次」可复用）。
- 可选：P 暂停/继续。

### 5.3 流程

1. 打开页面 → 显示「开始游戏」。
2. 用户按键开始 → 蛇按固定间隔移动，可控制方向，吃食物加分、长身体。
3. 蛇头撞墙或撞自身 → 游戏结束，显示分数与「再玩一次」。
4. 用户选择再玩一次 → 回到步骤 2 的初始状态。

## 6. 可选扩展

- 使用 localStorage 记录并显示历史最高分。
- 多个难度档位（不同初始速度或加速度）。
- 简单音效：吃食物、游戏结束（需说明浏览器自动播放策略）。
- 暂停/继续（P 键或按钮）。
- 触屏设备上的虚拟方向键（可选）。
